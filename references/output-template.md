# 能力产品方案输出模板

根据证据状态选择一个标题：

- `# 能力产品方案`：满足正式产品条件。
- `# 能力产品试卖方案`：能力证据存在但支付意愿待验证。
- `# 最小可测试能力产品`：关键条件仍主要依赖假设。

## 1. 当前判断

- 产品状态：正式产品 / 试卖产品 / 探索实验
- 证据等级：E0–E5
- 推荐理由：用两到四句话说明
- 最大不确定性：只列最关键的一至三项

## 2. 能力资产摘要

列出知识、技能、经历、资源、结果证据、外部认可和付费证据。没有证据的内容标注为自我评价。

## 3. 候选方向比较

最多三个方向。每个方向列出客户、问题、结果、产品形态、五维评分、事实、推断和待验证假设。

## 4. 推荐产品

- 一句话定位：为【特定客户】在【购买情境】解决【具体问题】，通过【交付机制】获得【可检查结果】。
- 目标客户：包括适合和不适合的人。
- 购买情境：问题何时发生，为什么现在需要解决。
- 结果承诺：与证据等级一致。
- 信任依据：用户已有的案例、经验和外部证据。

## 5. 交付设计

- 产品形态
- 交付步骤
- 周期与节奏
- 客户需要提供什么
- 包含项
- 不包含项
- 结果依赖和风险边界
- 完成交付的验收标准

| 升级价 |  |  |  |  |  |

价格缺少付费证据时，明确标注为测试假设。

## 7. 首批客户与名单

- 首批客户画像
- 名单来源
- 筛选问题
- 优先联系顺序
- 不应联系的人

## 8. 邀约与招募

### 私聊邀约

写成自然、低压力、允许拒绝的短消息，不虚构稀缺性和结果保证。

### 公开招募

包括适合人群、真实问题、交付方式、试卖条件、名额理由和报名动作。

## 9. 验证计划

- 需要验证的假设
- 需求访谈问题
- 试卖步骤
- 有效信号
- 成功指标
- 停止或调整条件

## 10. 事实、推断与假设

分别列出，不混写。

## 11. 未来七天行动

按日期或先后顺序列出具体动作、数量和完成标准。优先安排访谈、邀约和试卖，不安排重资产制作。
