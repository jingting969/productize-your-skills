# 能力产品化框架

## 核心原则

- 从具体经历和外部证据出发，不把兴趣或自我评价直接当成产品。
- 先确定客户的购买情境和问题，再设计交付内容。
- 用户自身证据优先于案例库中的成功结构。
- 先试卖和验证，再投入课程、网站或大规模营销物料。
- 始终分开标注事实、推断和待验证假设。

## 对话阶段

1. 建立目标：确认个人业务目标、可投入时间和已有方向。
2. 广泛盘点：扫描知识、技能、经历、资源、兴趣和他人求助。
3. 证据提取：追问项目、行动、产出、前后变化、反馈、付费和复购。
4. 候选方向：最多给出三个方向，写明客户、问题、结果、形态、证据和假设。
5. 比较选择：应用五维评分；用户选择，或由 Skill 给出有理由的推荐。
6. 产品设计：确定定位、承诺、交付、周期、客户配合、包含项和不包含项。
7. 三级定价：生成试卖价、标准价、升级价和涨价触发条件。
8. 首次验证：生成客户画像、名单来源、邀约、访谈、试卖步骤和验证指标。
9. 分级交付：输出正式产品、试卖产品或探索实验。

## 提问规则

- 每轮只问一个最关键的问题。
- 优先给出互斥选项，并允许自由补充。
- 不重复询问已经获得的信息。
- 每个阶段结束时回显关键结论并请用户确认。
- 若用户已有成熟方向，跳过广泛盘点，直接审查证据和产品结构。
- 信息不足时停止补全，转为假设和验证任务。

## 产品定义

每个方向都必须回答：

1. 谁在什么情境下购买？
2. 他正在承受什么具体问题？
3. 产品承诺什么可检查的产出或变化？
4. 为什么用户有能力交付？
5. 通过什么步骤、周期和客户配合交付？
6. 包含什么，不包含什么？
7. 为什么现在值得付费验证？

## 三级定价

- 试卖价：降低首次决策风险，交换真实付费、反馈、案例和交付学习。
- 标准价：交付流程和基础需求得到验证后使用。
- 升级价：通过更深服务、更快交付、更多支持或更高确定性提高价值。
- 每档价格都说明适用对象、差异、价值锚点和涨价条件。
- 不仅按工时定价，也考虑结果价值、替代成本、风险、速度和支持强度。
- 未经付费验证的价格必须标注为测试假设。

## 首次销售验证

至少输出：首批客户画像、名单来源、筛选条件、私聊邀约、公开招募、访谈问题、试卖步骤、成功指标和停止条件。

有效信号按强度递增：愿意详细描述问题、接受进一步沟通、提供内部信息、支付订金、完成购买、转介绍、复购。点赞和泛泛称赞不是购买证据。

## 防错清单

- 不把兴趣直接包装成产品。
- 不把“别人说不错”当作支付意愿。
- 不把目标客户定义为所有人。
- 不让结果承诺超过证据等级。
- 不用增加交付内容掩盖定位和边界不清。
- 不只按工时成本定价。
- 不在访谈和试卖前开发完整课程。
- 不因参考成功案例而忽略用户的资源和信任基础。
- 不只输出漂亮文案而缺少验证动作。
