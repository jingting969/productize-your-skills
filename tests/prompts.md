# Forward-Test Prompts

## Prompt 1：完全不知道卖什么

> 我工作八年，做过运营、写作、社群和项目管理，但我不知道自己能卖什么。

期望行为：

- 不立即生成产品方案。
- 每次只问一个问题。
- 从经历、结果、外部认可和付费证据开始盘点。
- 在证据不足时明确准备输出最小可测试产品。

## Prompt 2：已有方向但没有付费

> 我经常免费帮朋友改简历，至少十个人拿到了更多面试，但没人付过钱。我想做简历服务，帮我设计产品和价格。

期望行为：

- 识别至少 E2 结果证据，但不声称支付意愿已验证。
- 输出试卖产品，而不是正式产品。
- 设计试卖价、标准价、升级价，并把价格标成测试假设。
- 给出首批客户、邀约和付费验证动作。

## Prompt 3：范围过大的产品

> 我想帮助所有普通人提升人生，做一个包含课程、社群、咨询、工具和线下活动的产品。

期望行为：

- 阻止“所有人”定位和过重首版。
- 追问具体客户、购买情境和问题。
- 建议单一核心交付和轻量试卖，而不是继续堆内容。

## Prompt 4：案例学习

> 这是我设计好的产品，请拆解后加入案例库：面向独立咨询师的 14 天获客冲刺，价格 2999 元，包含定位、名单和私聊反馈。

期望行为：

- 先询问来源和保存、引用许可。
- 按案例模板拆解，区分缺失资料。
- 提取可迁移结构和不可照搬条件。
- 获得许可后才建议写入案例库和更新索引。

## Prompt 5：照抄案例风险

> 案例库里哪个产品卖得最好？直接照着给我做一个一样的。

期望行为：

- 不虚构销量或“最好”。
- 不直接照抄案例。
- 先检查用户能力证据、客户和问题，再说明可借鉴结构。
